# Phone Call with ElevenLabs TTS

Place a phone call to a French number using Twilio and play text-to-speech audio generated by ElevenLabs.

## Prerequisites

The following environment variables MUST be set before executing. Use AskUser to confirm they are available:

| Variable | Description |
|----------|-------------|
| `TWILIO_ACCOUNT_SID` | Twilio Account SID |
| `TWILIO_AUTH_TOKEN` | Twilio Auth Token |
| `TWILIO_FROM_NUMBER` | Twilio phone number (E.164 format, e.g. `+1415...`) |
| `ELEVENLABS_API_KEY` | ElevenLabs API key |

## Workflow

### Step 1 — Gather parameters

Use AskUser to collect:
- **Phone number**: Must be a French number in E.164 format (`+33XXXXXXXXX`). Validate the format: must start with `+33`, followed by 9 digits. Strip any leading `0` after the country code if the user provides local format (e.g. `06 12 34 56 78` → `+33612345678`).
- **Message text**: The text to speak. Default language is French.
- **Voice**: ElevenLabs voice ID. Default to `pFZP5JQG7iQjIQuC4Bku` (Lily, a natural French female voice). Suggest options: `Lily (French female)`, `Henri (French male)`, or `custom voice ID`.

### Step 2 — Verify environment

Run a quick check to confirm all 4 env vars are set (zsh-compatible):

```bash
for var in TWILIO_ACCOUNT_SID TWILIO_AUTH_TOKEN TWILIO_FROM_NUMBER ELEVENLABS_API_KEY; do
  eval val=\$$var
  if [ -z "$val" ]; then echo "MISSING: $var"; else echo "OK: $var"; fi
done
```

If any are missing, tell the user and stop.

### Step 3 — Generate TTS audio with ElevenLabs

Use the ElevenLabs text-to-speech API v1. Write the audio to a temp file.

```bash
curl -s -X POST "https://api.elevenlabs.io/v1/text-to-speech/${VOICE_ID}" \
  -H "xi-api-key: ${ELEVENLABS_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "'"${MESSAGE_TEXT}"'",
    "model_id": "eleven_multilingual_v2",
    "voice_settings": {
      "stability": 0.5,
      "similarity_boost": 0.75,
      "style": 0.0,
      "use_speaker_boost": true
    }
  }' \
  --output /tmp/agentcd_tts_output.mp3
```

Verify the file was created and is > 0 bytes. If it fails, check the HTTP response for errors.

### Step 4 — Host the audio publicly

Twilio needs a publicly accessible URL to play audio via `<Play>`. Upload to catbox.moe (reliable, no auth required):

```bash
AUDIO_URL=$(curl -s -F "reqtype=fileupload" -F "fileToUpload=@/tmp/agentcd_tts_output.mp3" https://catbox.moe/user/api.php)
```

Verify `AUDIO_URL` is a valid URL (should start with `https://files.catbox.moe/`). If catbox.moe fails, try these fallbacks:

**Fallback A — file.io:**
```bash
AUDIO_URL=$(curl -s -F "file=@/tmp/agentcd_tts_output.mp3" https://file.io | python3 -c "import sys,json; print(json.load(sys.stdin)['link'])")
```

**Fallback B — ngrok (if installed):**
```bash
python3 -m http.server 8765 --directory /tmp &
ngrok http 8765
# Use the ngrok HTTPS URL + /agentcd_tts_output.mp3
```

Store the resulting URL in `AUDIO_URL`.

### Step 5 — Place the call via Twilio

**IMPORTANT:** Use Python for the Twilio API call — shell variable expansion with `curl` and `${}` is unreliable in this context. Use `<Play>` with the hosted audio URL, NOT `<Say>` (which can inject unwanted prompts).

```python
python3 -c "
import os, urllib.request, urllib.parse, json, base64

sid = os.environ['TWILIO_ACCOUNT_SID']
token = os.environ['TWILIO_AUTH_TOKEN']
from_num = os.environ['TWILIO_FROM_NUMBER']

url = f'https://api.twilio.com/2010-04-01/Accounts/{sid}/Calls.json'
twiml = '<Response><Pause length=\"1\"/><Play>${AUDIO_URL}</Play></Response>'

data = urllib.parse.urlencode({
    'To': '${PHONE_NUMBER}',
    'From': from_num,
    'Twiml': twiml
}).encode()

credentials = base64.b64encode(f'{sid}:{token}'.encode()).decode()
req = urllib.request.Request(url, data=data, method='POST')
req.add_header('Authorization', f'Basic {credentials}')

try:
    resp = urllib.request.urlopen(req)
    result = json.loads(resp.read())
    print(json.dumps(result, indent=2))
except urllib.error.HTTPError as e:
    print(f'Error {e.code}: {e.read().decode()}')
"
```

Replace `${AUDIO_URL}` and `${PHONE_NUMBER}` with actual values inline in the Python string.

**Why not `<Say>`:** The `<Say>` TwiML verb can inject unwanted audio prompts (e.g. "press any key to execute your code") depending on account configuration. Always use `<Play>` with a hosted audio file for reliable message delivery.

### Step 6 — Report result

Parse the JSON response. Report to the user:
- Call SID
- Call status
- Any errors (e.g. invalid number, insufficient funds, geo-permissions)

Common Twilio errors for French numbers:
- **21215**: Geo-permission not enabled for France. User must enable it in Twilio Console → Voice → Geo Permissions.
- **21214**: Invalid `To` number. Check E.164 format.
- **20003**: Authentication error. Check SID/token.

## Important Notes

- French numbers: always E.164 format `+33XXXXXXXXX` (9 digits after country code)
- ElevenLabs `eleven_multilingual_v2` model handles French natively
- Twilio requires France to be enabled in Voice Geographic Permissions
- **Always use `<Play>` with hosted audio, NEVER use `<Say>`** — `<Say>` can inject unwanted system prompts
- **Always use Python** for the Twilio API call — shell `${}` variable expansion is unreliable with `curl` in zsh
- **Use catbox.moe** for audio hosting — file.io and 0x0.st are unreliable (empty responses, blocked MIME types)
- Always clean up temp files after the call is placed
- Never log or display API keys in output
